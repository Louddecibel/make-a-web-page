Lesson 1

The Web and Accessing it

A lot of things are happening in the background when you do something as simple as accessing a Web Page. When you go to your favourite website using your browser, it uses a language called HTML ,which stands for Hyper Text Mark Up Language, that tells your browser how to display contant and what happens when the user interacts with it.
When you type an address into your browser, your computer is essentially fetching files from a server half way across the world for your viewing pleasure. But how does it do this? Using HTTP, Hypertext Tranfer Protocal, a protocal used to communicate with servers.

HTML Structure

I like to think of HTML like the Human body; It has an element on top that defines what type it is (DOCTYPE), a head with a name and a brain that contains Metadata defining how it should communicate with you (Title, CSS, JS), and body that has all the contant you want to consume (Images, videos, text). It looks a little like this:

DOCTYPE--->defines it as HTML
Head--->Contains the name of the page and other Data linking to CSS and JS
Body--->That contant you are here for!

Elements

Elements contains Tags that tell your browser how it should display contant. These are used to display links, images, and format contant to look a certain way. So far, we know that there are two type of elements in HTML; inline and block elements.
Inline elements will display contant on the same line. However, block elements will create an invisable box around contant giving it a block of its own (just like how this page is divided into seperate section using the DIV block element).

Lesson 2

What is HTML and CSS?

HTML: is the structure every website is built on.
CSS: determines the way a website will look visualy...kinda like a stylist.

Language, syntax, and DOM

To really understand how a website works, you need to know that HTML is uses a branching structure called DOM(Document object model). When you write HTML code, with its own set of syntax and rule that usually consist of tags, you're telling your browser how to display things. And through DOM, it identifies the syntax and rules in the code in order to correctly display your webpage. Additionally, CSS has its own language, syntax and rules that determines visual and style changes to your website.

Boxifying

A website is made of boxes that has all the images, text, headers, footers, you name it! When you come to terms with that, it becomes easier to layout your webpage, think about the elements you want to put in it, and begin designing and building your own webpage.

Lesson 3

Define CSS

Cascading Style Sheet (CSS) defines the way a website will appear visually. It is a separate document that is linked to the HTML document of the website that not only changes the color and style of your elements, but can also define the layout.

Why CSS?

There is a couple of advantages for using CSS:

Stops repeatition: Instead of coding multiple elements in your HTML documents with the same styling, you can set a class for all of elements that you know will have the same styling and set the rules for them in CSS to avoid repetition.

Less mistakes and easier to review: Coding multiple elements in your HTML document with the same styling will inevitably result in some error. Using a single class definition in your CSS document to set rules for several elements in HTML will have a smaller margin of error. Better yet, if something is visually wrong in your website, it easier to pinpoint the culprit using CSS.

Using CSS

There are three main methods for applying styling to your HTML document.

  1. It can be done within the HTML documents head by placing CSS rules within style tags
  2. they can be added inline within each tag
  3. or in a separate CSS file linked to the HTML document

CSS follow the “inheritance” rule where the most specific rule is applied. This means that child elements within an element with a defined CSS rule will inherit the values of its ancestor. However, not all rules are inherited and the child element can override these values by setting and defining CSS rules for it.

ID vs Class

Class is used to style several container elements that are related it someway (headers, titles, footers), while ID’s are used to style a unique element within your HTML document.

The Box Model

It easier to understand the layout of your website and its elements once you look at the “Box Model.” Each element in your HTML document has a value (usually the content), padding, a border, and a margin. Change these values cause change to the space, or the box, your element occupies but not to the layout of the page.

Flex and Responsive Design
